What does javascript event loop do?

The event loop works between the call stack and the callback queue. The event loop determines whether any functions are still waiting to be executed in the call stack. If the call stack has any functions left to be executed, it simply runs the functions, and once the function is executed, the function is automatically removed from the call stack. If there is no function except main(), then the event loop loops itself for one time and calls the functions that stay on the callback queue, and sends those functions into the call stack. This is how an event loop works in JavaScript.

Difference between local storage vs session storage:

  • Local storage data does not expire if the browser, PC, and tab are closed, whereas session storage data is removed after the page session expires. When a page is loaded into a browser tab, a new session is generated.
  • Local storage has a storage capacity of 5MB/10MB, whereas session storage has a storage capacity of 5MB.
  • Local storage data must be erased using JavaScript or manually because it is not session-based. Session storage works per tab or window since it is session-based.
